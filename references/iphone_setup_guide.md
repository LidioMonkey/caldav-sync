# iPhone / iPad CalDAV 日历连接配置指南

## 前提条件

1. CalDAV 服务已部署并通过 HTTPS 访问
2. 已创建用户账号和密码
3. iPhone 已连接互联网

## 步骤详解

### 第一步：打开设置

1. 打开 iPhone **设置** App
2. 向下滚动，找到 **日历**
3. 点击 **账户**

### 第二步：添加账户

1. 点击 **添加账户**
2. 在账户类型列表中，向下滚动到最底部
3. 选择 **其他**
4. 在「日历」部分，点击 **添加 CalDAV 账户**

### 第三步：填写服务器信息

| 字段 | 填写内容 | 说明 |
|------|----------|------|
| **服务器** | `cal.example.com` | 你的域名，**不要加** `https://` |
| **用户名** | `yourname` | CalDAV 用户名 |
| **密码** | `yourpassword` | CalDAV 密码 |
| **描述** | `个人日历` | 随意填写，用于在列表中识别 |

### 第四步：高级设置（可选但推荐）

填写完毕后，点击 **高级设置**：

| 字段 | 值 |
|------|-----|
| 使用 SSL | ✅ **开启** |
| 端口 | `443` |
| 账户 URL | `https://cal.example.com/cal.php/principals/yourname/` |

> **重要**：账户 URL 的路径格式为 `/cal.php/principals/<用户名>/`，这是 Baikal 的固定格式。

### 第五步：验证并保存

1. 点击右上角 **下一步**
2. iPhone 将验证服务器连接
3. 验证通过后，选择要同步的内容（日历、提醒事项）
4. 点击 **存储**

### 第六步：确认同步

1. 打开 **日历** App
2. 点击底部的 **日历** 按钮
3. 确认新添加的日历账户已勾选
4. 等待几秒钟，日历数据开始同步

## 验证连接是否成功

在 iPhone 上验证：

1. **设置 → 日历 → 账户** → 点击你的 CalDAV 账户
2. 确认状态显示正常（无错误提示）
3. 查看「上次同步」时间

在服务器上验证：

```bash
# 检查 Nginx 访问日志，确认来自 iPhone 的请求
sudo tail -f /var/log/nginx/caldav_access.log | grep "iPhone\|iOS\|CalDAV"
```

## 多设备同步

所有设备使用**完全相同的账户信息**即可自动同步：

| 设备 | 设置路径 |
|------|----------|
| iPhone | 设置 → 日历 → 账户 |
| iPad | 设置 → 日历 → 账户 |
| Mac | 系统设置 → 互联网账户 → 添加其他账户 → CalDAV |
| Thunderbird | 新建日历 → 网络上 → CalDAV |

## 故障排查

### "无法验证账户信息"

1. 确认服务器地址**不带** `https://` 前缀
2. 确认用户名和密码正确
3. 尝试在 Safari 中访问 `https://你的域名/` 确认服务器可达
4. 确认 SSL 证书有效（不是自签名证书）

### "无法使用 SSL 连接"

1. 确认使用了 Let's Encrypt 等受信任的 CA 颁发的证书
2. iPhone 不信任自签名证书 — 必须用正式 CA 证书
3. 在 Safari 中打开 `https://你的域名/`，看是否有证书警告

### 日历事件不同步

1. 确认在「日历」App 底部勾选了该日历
2. 尝试下拉刷新日历列表
3. 在「设置 → 日历 → 账户」中关闭再重新开启日历同步
4. 检查服务器磁盘空间是否充足

### 同步速度慢

1. CalDAV 是增量同步协议，首次同步大量事件可能较慢
2. 确保服务器和 iPhone 网络连接稳定
3. 检查服务器 CPU/内存使用情况

## CalDAV URL 格式参考

Baikal 的 CalDAV 路径结构：

```
主端点：       https://<domain>/cal.php/
用户主目录：    https://<domain>/cal.php/principals/<username>/
默认日历：     https://<domain>/cal.php/calendars/<username>/default/
```

iPhone 的「账户 URL」应填用户主目录路径。
